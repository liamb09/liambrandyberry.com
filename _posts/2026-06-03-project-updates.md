Last week I was working on my physics engine more, but I’ve shifted to something new that I’ve been really excited about. Even one week into summer I’m just barely hanging on to my goal to stick to longer-form projects. I don’t think this is necessarily a bad thing, though, because this new project is something very different from what I normally do and it’s still significant, so I think it will be a good learning experience.

Basically what I’m doing is creating a website through which people can order sourdough bread from me. It’s a great intersection between two of my hobbies, so I’ve been enthusiastically working on it lately. So far I’ve built up most of the actual website; I have a main page listing the loaves I sell, a page to add each item to the bag, and a bag page that people will soon be able to order from. 

All of this is not too different from what I’ve done in the past, so the really interesting part of the site is how I’m planning to host it—I can’t do it through GitHub Pages because it’s not static, so right now I’m planning to host it on my old laptop. I’ve looked a bit into it, and it seems like it shouldn’t be too difficult to install a server and use Cloudfare Tunnels so that I can host the shop at ``bread.liambrandyberry.com``.  This way, I can receive orders on the laptop and push notifications via email or text. This is something I’ve never worked with, so I’m really excited to try it out. I also think my OS knowledge from CyberPatriot will come in handy here, so it will be a nice all-around project.

I’m aiming to have the site ready and operational in the next couple weeks so I can hopefully fulfill some orders this summer, although it will have to be very small-scale as my time will be pretty restricted and sourdough breads take a couple days of work in advance.
